# SpawnWP 0.3.12

This release makes first-run two-factor enrollment more forgiving and easier to
troubleshoot.

- The setup authenticator check now accepts a code within ±60 seconds instead of ±30,
  so a server whose clock has drifted slightly no longer rejects every TOTP code.
- Password-plus-TOTP fallback sign-in uses the same wider window; the single-use
  replay protection is unchanged.
- When a setup code is rejected, the message now explains the two real causes — an
  out-of-sync server clock and authenticator apps that ignore the SHA-256 algorithm
  (or a secret typed in by hand) — and how to fix each.
