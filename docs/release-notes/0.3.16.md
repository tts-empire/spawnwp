# SpawnWP 0.3.16

This release extends the opt-in telemetry so performance problems (like the slow
creates fixed in 0.3.13) can be spotted proactively instead of waiting for a user
report.

- **Telemetry notice v3.** Installations that consent under the new notice also
  share three groups of strictly aggregate data:
    - *Create performance*: warm/cold provisioning durations (count, sum, worst),
      total and failed creates, healthcheck timeouts.
    - *Feature usage*: blueprint counts, temporary sites created and auto-expired,
      creates with custom PHP settings, destroys, PHP switches, image refreshes and
      deletes.
    - *Machine specs (rounded)*: CPU count, RAM, total/free disk, Docker image and
      build-cache footprint, number of PHP images.
- **Gradual rollout, no re-prompt.** Existing consents given under notice v2 keep
  sending exactly the minimal payload they agreed to until their natural 90-day
  renewal. The privacy guarantees are unchanged: never domains, IPs, email,
  usernames, site names, content, plugins, logs or credentials. The
  [privacy notice](https://spawnwp.com/privacy/telemetry) is updated accordingly.
- **Local counters regardless of consent.** The aggregate counters are collected in
  `/var/lib/spawnwp/metrics.json` on every installation (atomic, best-effort, never
  blocking an operation) — they never leave the machine without a v3 consent, and
  they lay the groundwork for future cockpit statistics.
- **UI polish.** The badge on the Deploy PHP settings panel now reads
  **"N custom values"** (for example "3 custom values") instead of the ambiguous
  "modified".
