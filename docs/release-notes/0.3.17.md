# SpawnWP 0.3.17

A small release that removes the last "first-time" surprise.

- **PHP image pre-build in the installer.** The installer now asks
  `Pre-build the PHP image now so the first deploy is fast? [Y/n]` (default: yes)
  right after the other questions, and runs the build at the very end of the
  installation. The trade-off is stated up front: about 5 extra minutes of install
  time and ~1.8 GB of disk, in exchange for a **~35-second first site creation**
  instead of a one-off ~5-minute image build. Skipping keeps the previous
  behaviour, with the cockpit's clear first-build notice. The final installation
  report states which case applies.
- Scripted installations can pre-seed the choice with `PREBUILD_PHP_IMAGE=0` (skip)
  or `=1` in the environment, like the other installer prompts.
- A pre-build failure never fails the installation: a warning is printed and the
  first site creation builds the image as usual.
