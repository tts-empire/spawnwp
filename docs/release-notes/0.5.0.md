# SpawnWP 0.5.0

Every SpawnWP site has always shipped [WP-CLI](https://wp-cli.org/) inside its PHP
container — until now you needed an SSH session on the host to use it. This release
puts it one click away: a **WP-CLI console** on every site card in the cockpit.

## The WP-CLI console

Press **⌨ WP-CLI** on a site card, type a command — with or without the leading
`wp` — and press Enter:

- the command runs inside that site's PHP container as the web user;
- the output streams live into the card's output box, so long jobs like
  `wp media regenerate` or a large `wp search-replace` show their progress line by
  line;
- ↑/↓ recall earlier commands from the session;
- commands registered by installed plugins work too (`wp wc ...` with WooCommerce
  active, and so on).

## How it runs, and what that means

Each run is a **single, non-interactive `wp` process** — the cockpit executes an
argument vector, no shell is involved and nothing runs on the host. A few practical
consequences:

- **Interactive subcommands are rejected** with an explanation: `wp shell`,
  `wp db cli` and `--prompt` need a real terminal. Passing input as arguments works:
  `wp db query "SELECT ..."` is fine.
- **Confirmations need `--yes`.** Commands that normally ask "Are you sure?" cannot
  prompt here: they abort safely without doing anything, and the console reminds you
  to add `--yes`.
- **No shell operators.** `|`, `>` and `&&` have no effect — use WP-CLI's own
  `--format=csv` / `--format=json` / `--field=...` options and copy the output.
- **Files must already be inside the container.** For `wp db import` and friends,
  drop the file into the site's `wp-content` bind mount on the host
  (`/srv/<site>/projects/primary/wp-content/`).

The console is not a sandbox — it is full WP-CLI on your own disposable site,
confined to that site's container and protected by the cockpit session. See
[Using the cockpit](../using-the-cockpit.md#the-wp-cli-console) and the note in
[Security](../security.md).

## Under the hood

- New cockpit endpoint `POST /api/wp-cli/{project}` reusing the existing SSE
  streaming pipeline; command lines are parsed with POSIX quoting rules and
  validated before execution.
- Telemetry (consented installs only): new aggregate counter `wp_cli_commands`.
- No migration required; existing installations get the feature with the regular
  update.
