# SpawnWP 0.5.1

A small polish release for two rough edges around authentication and updates.

## Fixes

- **Dashboard updates no longer look like failures.** While the cockpit restarts
  itself during an update, the Updates page used to surface the temporary
  connection drop as a red `JSON.parse` error — even though the update had
  succeeded. The restart window is now handled as a normal "reconnecting" phase;
  an error is shown only when the updater itself reports a failure, and the page
  confirms the new version as soon as the cockpit is back.
- **Login setup can be retried.** If the authenticator code was rejected during
  first enrollment, clicking *Verify code and create passkey* again crashed with
  `v.replace is not a function`. The passkey ceremony options are now cloned per
  attempt, so you can simply re-enter the code.
- Internal: the authentication test suite is hardened to always operate on its
  temporary database, independent of test import order.

No migrations, no behavior changes to sites or blueprints.
