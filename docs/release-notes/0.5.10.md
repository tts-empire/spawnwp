# SpawnWP 0.5.10

This is a fix release for the per-site **file browser** introduced in 0.5.9.

## Fix: cockpit startup after 0.5.9

The 0.5.9 upload endpoint accepted the file as multipart form data, which FastAPI can only
serve when the `python-multipart` package is installed. That package is not part of the
cockpit environment, so FastAPI raised at route-registration time and the cockpit service
failed to start after updating to 0.5.9.

The upload endpoint now streams the file in as the **raw request body**, with the target
folder and filename passed as query parameters. This removes the `python-multipart`
dependency entirely — nothing extra to install — and the file browser (browse, view, edit,
download, upload, new folder, rename, delete) works as intended.

## Notes

- If your cockpit is on 0.5.8 or earlier, updating straight to 0.5.10 is safe; you never
  need 0.5.9. A failed 0.5.9 update leaves the previous version running, so there is no
  downtime — updating to 0.5.10 completes the upgrade.
- No behaviour change to any other endpoint, and no change to the Deploy plugin.
