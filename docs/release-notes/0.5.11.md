# SpawnWP 0.5.11

A small usability fix for the per-site **file browser** added in 0.5.9/0.5.10.

## One scrollbar, not two

The file browser renders inside a site card's output box, which is itself a scrollable,
height-capped area. That meant the browser's own content added a second vertical scrollbar
right next to the box's — two near-identical bars side by side, awkward to use, most
noticeably when editing a file (the editor's textarea scrolled inside the capped box).

Now the inner content owns the scrolling:

- **Folder list.** The list scrolls in its own area under a **fixed breadcrumb**, so you can
  always see and click where you are while a long folder scrolls.
- **File editor.** The editor's textarea is the only thing that scrolls.

Other panels in the same card (PHP settings, WP-CLI, disk usage, logs) are unchanged.

## Notes

- Cockpit-only change (assets); no backend, dependency or Deploy-plugin changes.
