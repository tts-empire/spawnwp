# SpawnWP 0.5.13

Administrator activation no longer gets stuck on machines without a built-in passkey, and the
passkey prompts are more forgiving and better explained.

## Activate with password + code, add a passkey later

The one-time administrator activation used to **require** creating a passkey to finish. On a
computer with no platform authenticator — for example a Windows PC where Windows Hello/PIN
isn't set up — the browser could only offer a phone or a security key, and if that flow was
cancelled or timed out the setup ended with *“The operation either timed out or was not
allowed.”* — with no way past it.

Now the passkey step is **optional**:

- On the setup screen a **Skip — activate with password + code** button completes enrollment
  with your password and authenticator (TOTP) code. You still receive recovery codes, and
  sign-in keeps working with password + code.
- After signing in, the **Manage** page shows a one-click **Add a passkey** banner so you can
  add one whenever a suitable authenticator is available (this device's PIN/biometrics, your
  phone, or a security key). The banner disappears once a passkey is registered.

Passkeys remain the recommended, phishing-resistant sign-in — this only removes the hard
requirement at activation time.

## A sturdier passkey flow

- **Longer timeout.** WebAuthn ceremonies now allow 180 seconds (was 60), enough time for the
  cross-device (phone) flow during first-run setup.
- **Up-front hint.** The setup screen detects when a device has no built-in passkey and tells
  you what to expect (use a phone or security key, or skip and add one later).
- **HTTPS guard.** Passkey actions stop early with a clear message if the page isn't a valid
  secure (HTTPS) context, instead of failing deep in the browser.
- **Better error messages.** Cancel/timeout, missing Windows Hello, an already-registered
  passkey, or an unsupported browser now produce specific guidance rather than the raw
  browser error string.

## Notes

- Cockpit-only change (`auth.py`, the login page, and the Manage banner). No Deploy-plugin,
  dependency, or on-disk data changes; existing administrators, passkeys and sessions are
  unaffected.
