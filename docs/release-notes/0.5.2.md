# SpawnWP 0.5.2

A focused cockpit polish release that makes blueprint ownership and origin explicit.

## Cockpit polish

- **Blueprint origins are clearer on Deploy.** Built-in choices appear under
  **SpawnWP blueprints**, captures made with the SpawnWP Deploy plugin appear under
  **Your blueprints**, and advanced schema v1 files appear under **Custom manifests**.
  The ambiguous `Template` badge is no longer shown. When there are no captured
  blueprints yet, the page links to **System → Template connections**.

No migrations or changes to provisioned sites, blueprint manifests or payloads.
