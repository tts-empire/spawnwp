# SpawnWP 0.5.21

One change, closing a version gap.

## The Deploy plugin checkbox installs the latest release

The cockpit's **Install the SpawnWP Deploy plugin** checkbox used to install a zip bundled into
the SpawnWP core release. The plugin releases on its own cadence, so that bundled copy went stale
the moment a new plugin version was published between two core releases — and a freshly spawned
site could start life one or more plugin versions behind, with no way to know it.

Now, when the checkbox is ticked, SpawnWP asks spawnwp.com for the newest published plugin
release and — only if it is **strictly newer** than the bundled copy — downloads it and verifies
it end to end: the SHA-256 checksum first, then the Ed25519 signature against the plugin release
key, which now ships with the runtime. The version inside the zip must also match what the
manifest claimed. If any of that fails, or the server has no internet access, the spawn simply
uses the bundled copy as before — the fetch can delay a site's creation by a few seconds, never
break it. The deploy log states which version was installed and whether it came from spawnwp.com
(verified) or the bundle.

Nothing changes for the plugin itself or for manual installs from the
[download page](https://spawnwp.com/plugin/); the artifacts and their signatures are the same ones
published there.
