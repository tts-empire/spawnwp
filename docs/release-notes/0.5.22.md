# SpawnWP 0.5.22

Three improvements to the cockpit, all from the same piece of user feedback.

## Snapshots can be named and deleted

A snapshot used to be a timestamp and nothing else: no way to tell which one you took before
switching themes, and no way to remove the ones you no longer needed. Both are fixed.

Double-click a snapshot — or use the pencil button — to give it a name, and the 🗑 button removes
it along with its database dump and its uploads tarball. Deleting a snapshot destroys a restore
point, so it asks for the same security confirmation as a restore.

The snapshot files keep their timestamp filenames; the names live in a small `labels.json`
alongside the backups. That is deliberate: the timestamp is what protects the restore path from
being talked into reading a file elsewhere on disk, and renaming files would have put that
guarantee in the hands of whatever you typed into the name box. Snapshots taken before this
release simply show up without a name until you give them one.

## The file manager takes folders and zip archives

Uploading anything with a directory structure meant creating each folder by hand and then
uploading files one at a time. Now **⬆ Upload folder** takes a whole tree and rebuilds it on the
site, and any `.zip` in the listing gets a 📦 button that extracts it in place.

Archives are inspected before anything is written: if any entry would land outside the folder
you are extracting into — the classic "zip slip" — SpawnWP refuses the whole archive and tells
you which entry was the problem. Archives that are absurdly large or contain absurdly many files
are refused too, so a bad upload cannot fill the disk.

## Magic login, off by default

Getting into a site's wp-admin meant copying a username, then copying a password, then pasting
both. There is now a **🔑 Magic login** button next to the credentials that opens wp-admin
already signed in.

It is off on every site until you switch it on, because it is exactly what it sounds like: a way
into WordPress that skips the login form. Enabling it installs a small must-use plugin; disabling
it removes that plugin, and with it any way to use the feature.

The links are single-use and expire in two minutes. SpawnWP never stores the link's secret — only
its SHA-256 — and the must-use plugin invalidates a link *before* it signs anyone in, so the same
link can never be used twice even by two requests arriving at the same moment. Magic login is
also excluded from Deploy packages, so it stays on the site where you enabled it and never
travels to a destination site.
