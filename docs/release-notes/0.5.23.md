# SpawnWP 0.5.23

A follow-up to 0.5.22, which shipped magic login switched off everywhere.

## New sites get magic login automatically

In 0.5.22 the feature had to be enabled on each site before it could be used, which rather
defeated the point: a one-click sign-in you have to set up per site is a one-click sign-in
to nothing. New sites now come with it ready to use.

Nothing changes for sites you already have — press **Enable magic login** on any of them
once and the button lights up; they do not need recreating. Turning it off still deletes
the must-use plugin from the site, so the capability disappears rather than being flagged
off somewhere.

If you would rather spawn sites without it, set `SPAWNWP_ENABLE_AUTOLOGIN=0` in
`/etc/spawnwp/config.env`.

The security properties are unchanged from 0.5.22: links are single-use, expire in two
minutes, can only be minted from an authenticated cockpit session, and the server stores
only their SHA-256, never the secret. Deploy packages still exclude `mu-plugins`, so magic
login never travels to a destination site.

## The documentation caught up

0.5.22 added three visible features and documented none of them. The cockpit guide now
covers naming and deleting snapshots, uploading folders, extracting zip archives — including
why an archive can be refused outright — and magic login end to end. The security reference
gains a section setting out exactly what a magic link is and every bound placed on it.
