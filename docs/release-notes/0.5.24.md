# SpawnWP 0.5.24

This release clarifies the Deploy page without changing blueprint or site-creation
behaviour.

## Two steps that look like two steps

**Choose a blueprint** and **Configure & create** now have prominent headings, numbered
accent badges and a visual divider. The previous 12-pixel muted labels were too close to
the page's secondary field labels and did not communicate the workflow clearly.

The hierarchy change is limited to Deploy; section labels elsewhere in the cockpit keep
their existing compact treatment.

## Captured-site guidance appears only when relevant

The warning about fresh administrator credentials appeared even with Clean, Demo,
Development and custom manifest blueprints. The selection code correctly applied the
HTML `hidden` state, but the panel's CSS forced it back to `display: grid`.

The panel now respects `hidden`, appears only for captured content blueprints, and says
exactly what is restored: content, settings, uploads, plugins and themes, but never source
user accounts or passwords. This was a presentation bug; the request sent when creating a
normal blueprint already ignored the captured-site plugin-deactivation option.
