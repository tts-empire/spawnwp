# SpawnWP 0.5.3

A cockpit-wide UI polish release focused on visual hierarchy, readability and
keyboard accessibility.

## System page

- PHP images, blueprint capture and storage now live in separate, consistent panels.
- **Generate pairing code** is a clear primary action with deliberate spacing.
- Connections and captured blueprints have their own subsections, while textual table
  actions use normal button styling instead of the icon-only treatment.

## Cockpit-wide polish

- The security-confirmation dialog is explicitly centered, handles short viewports
  safely and stacks its actions on narrow screens.
- Body copy, help text, tables, controls and labels follow a shared readable type scale.
- Keyboard focus is visible across buttons, links and form controls. Login and initial
  passkey/TOTP activation use the same control sizing and focus treatment.
- Motion is disabled for users who request reduced motion.

No migrations or changes to sites, blueprints, authentication APIs or update behavior.
