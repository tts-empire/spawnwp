# SpawnWP 0.5.6

Spawning a site from a **captured content blueprint** is now safer and less
surprising.

## Plugins start deactivated

A captured blueprint reproduces a full site, including its plugins and their
configuration. That can include security or login plugins — IP allow-lists,
passwordless login, brute-force lockouts — configured for the original site, which
would happily lock you out of the fresh copy.

So spawning from a captured blueprint now **deactivates every plugin by default**: the
plugins are installed but inactive, and a dismissible notice in the new site's
dashboard reminds you to **reactivate them one at a time**, so a problematic one is
easy to catch before it locks you out. A *Start with all plugins deactivated* checkbox
on the Deploy page is on by default; uncheck it to bring the site up with its plugins
active, exactly as captured.

## Clearer credentials

The Deploy page now says plainly what happens to accounts: a captured blueprint
creates a **fresh administrator** for the new site and never includes the source
site's user accounts or passwords. Log in with the new site's own credentials from
**Manage → 🔑 WP credentials**, not the source site's.

Built-in blueprints are unaffected. No migrations.
