Workflow

A guarded initial publication.

  1. 1

    Install on both sites

    Upload and activate the same plugin ZIP on the SpawnWP source and the fresh target.

  2. 2

    Connect for 15 minutes

    Generate a short-lived key on the target, paste it into the source and run preflight checks.

  3. 3

    Review and confirm

    The source transfers checksummed chunks. The target stages, verifies and activates only after confirmation.

  4. 4

    Health check or rollback

    The target preserves its URL and administrators, checks the result and automatically rolls back on failure.

Compatibility

Deliberately narrow.

Both sites must be WordPress single-site on the exact same core version and PHP major/minor version. PHP 8.1+, sodium, ZIP, trusted HTTPS, a reachable REST API, local uploads and direct filesystem writes are required.

The package limit is 2 GiB. WordPress core, users, wp-config.php, MU plugins, drop-ins and SpawnWP development tools are not copied.

Not supported: existing or live targets, repeated pushes, data merging, selective sync, Multisite or object-storage uploads.
