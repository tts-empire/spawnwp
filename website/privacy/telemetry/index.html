Privacy

Optional telemetry

Telemetry is optional, disabled by default and enabled only after an explicit installer prompt. Consent expires automatically after 90 days.

Collected

Operational metadata

A random installation identifier, event timestamp, SpawnWP version, operating system family and version, architecture, enabled feature flags and the aggregate number of current environments.

Excluded

No project identity or content

Payloads exclude domains, IP addresses, email, usernames, site names, WordPress content, plugins, logs and credentials.

Control

Inspect or disable

Run spawnwp telemetry status or spawnwp telemetry payload to inspect local state. Run sudo spawnwp telemetry disable to stop collection and delete the local identifier and pending queue.

Endpoint failures never block installation or cockpit operation. The implementation is available in the public source code.
